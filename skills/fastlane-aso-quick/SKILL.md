# Fastlane ASO Quick — 빠른 스토어 문구 실험

## 개요

심사 없이 즉시 반영 가능한 스토어 메타데이터만 다룬다:

- **iOS**: 프로모션 텍스트 (170자, 심사 불필요)
- **Android**: 짧은 설명 (80자)

목표: 최소 리드타임으로 문구 실험 사이클을 돌리는 것.

---

## 실행 흐름

### Step 1. 앱 정보 수집

사용자에게 아래 정보를 받는다:

- 앱스토어 또는 플레이스토어 URL (하나 이상)
- 타겟 로케일 목록 (기본: ko, en-US, ja)
- 현재 실험하고 싶은 방향이나 강조 포인트 (선택)

### Step 2. 앱 분석

웹 검색을 통해 다음을 파악한다:

1. **앱 현황 분석**
   - 현재 스토어 페이지의 프로모션 텍스트/짧은 설명 확인
   - 앱의 핵심 기능, 차별점, 타겟 유저 파악
   - 현재 카테고리 및 포지셔닝 확인

2. **경쟁 앱 분석**
   - 같은 카테고리 상위 앱 5개의 프로모션 텍스트/짧은 설명 조사
   - 어떤 키워드와 소구점을 쓰고 있는지 패턴 분석

3. **키워드 트렌드**
   - 해당 카테고리에서 현재 검색량 높은 키워드 파악
   - 시즌성/트렌드 키워드 확인

### Step 3. 문구 생성

분석 결과를 바탕으로 **각 로케일별로 3가지 버전**을 생성한다.

#### iOS 프로모션 텍스트 (최대 170자)

- 버전 A: 기능 중심 소구
- 버전 B: 감성/베네핏 중심 소구
- 버전 C: 시즌/트렌드 활용 소구

#### Android 짧은 설명 (최대 80자)

- 버전 A: 기능 중심 소구
- 버전 B: 감성/베네핏 중심 소구
- 버전 C: 시즌/트렌드 활용 소구

**생성 규칙:**

- 각 로케일은 번역이 아니라 해당 언어권 유저 관점에서 네이티브하게 작성
- 핵심 키워드를 자연스럽게 포함
- CTA(행동 유도) 요소 포함 권장
- 이모지 사용 여부는 카테고리/경쟁 앱 트렌드에 맞춤

**사용자에게 3가지 버전을 보여주고 선택받거나, 조합/수정 요청을 받는다.**

### Step 4. Fastlane 메타데이터 파일 생성

사용자가 선택한 문구를 Fastlane 메타데이터 구조에 맞게 파일로 생성한다.

#### iOS 디렉토리 구조

```
ios/fastlane/metadata/
├── ko/
│   └── promotional_text.txt
├── en-US/
│   └── promotional_text.txt
└── ja/
    └── promotional_text.txt
```

#### Android 디렉토리 구조

```
android/fastlane/metadata/android/
├── ko-KR/
│   └── short_description.txt
├── en-US/
│   └── short_description.txt
└── ja-JP/
    └── short_description.txt
```

**각 파일에는 선택된 문구만 들어가며, 파일 끝에 불필요한 줄바꿈을 넣지 않는다.**

### Step 5. 배포 명령어 제공

사용자에게 아래 명령어를 안내한다:

```bash
# iOS 프로모션 텍스트만 업로드 (심사 제출 없이)
cd ios
fastlane deliver --skip_binary_upload --skip_screenshots --force

# Android 짧은 설명만 업로드
cd android
fastlane supply --skip_upload_apk --skip_upload_aab --skip_upload_images --skip_upload_screenshots
```

**주의: 사용자에게 명령어 실행 전에 반드시 확인을 받는다. 자동 실행하지 않는다.**

### Step 6. 실험 기록

실험 추적을 위해 아래 형식으로 로그를 남긴다:

```markdown

| 2025-01-15 | iOS    | ko     | A    | 기능 중심 | 초기 테스트 |
```

## 주의사항

- iOS 프로모션 텍스트는 **심사 없이 즉시 반영**되므로 가장 빠른 실험이 가능하다
- Android 짧은 설명은 반영까지 수 시간 걸릴 수 있다
- 한 번에 너무 많은 요소를 바꾸면 어떤 변경이 효과를 냈는지 알 수 없다 → 한 번에 하나의 변수만 변경 권장
- 문구 변경 전 현재 지표(노출 수, 전환율)를 기록해두어야 비교 가능
- 앱 이름은 절대 변경하지 않는다
