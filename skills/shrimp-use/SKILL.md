# Shrimp Task Manager — 사용편

> 세션 간 메모리 유지 — 과거 작업 참조, 중복 방지.

---

## 행동 규칙

- **태스크 하나 = 파일 1~3개, 30분 이내 완료 가능한 크기** — 크면 과감하게 쪼갤 것
- **플랜 완료 후 반드시 멈출 것** — 태스크 목록 보고까지만. 실행은 절대 하지 않음
- **실행은 대표님이 계획 검토 후 직접 승인** — 승인 없이 execute_task 절대 금지
- **태스크 번호 언급 시 즉시 실행** — "3번", "#2" 같이 언급하면 확인 없이 바로 시작 (승인으로 간주)
- **완료된 태스크는 삭제 불가** — 장기 메모리로 영구 보존됨

---

## 태스크 분류 원칙

1. **인터페이스/계약/스키마 먼저** — 타입 정의, DB 스키마, API 계약은 구현보다 앞
2. **의존성 있는 것부터 순서 명시** — B가 A 결과물을 쓴다면 A → B 순서를 split_tasks에 명시
3. **독립적인 태스크는 병렬 표기** — 의존성 없으면 순서 강제 불필요

---

## 플랜 요청 시 문서 먼저

"플랜해줘 / 기획해줘 / 설계해줘" 요청이 오면 `plan_task` 실행 전 반드시 두 문서를 작성한다.

### 기존 문서가 있을 때 (재플랜 시만)

> "기존 PRD/Roadmap 문서가 있어요. 백업할까요?"

- **백업 O** → `docs/backup/YYYYMMDD_기능명/` 에 복사 후 새 문서 작성
- **백업 X** → 바로 덮어쓰기

### `docs/PRD.md`

```markdown
# PRD: [기능명]

## 개요
무엇을 왜 만드는가

## 사용자 스토리
- As a [user], I want to [action] so that [benefit]

## 기능 요구사항
- 핵심 기능 목록

## 비기능 요구사항
- 성능, 보안, 확장성 등

## 범위 외 (Out of Scope)
- 이번 버전에서 하지 않는 것
```

### `docs/ROADMAP.md`

```markdown
# Roadmap: [기능명]

## Phase 1 - MVP
- [ ] 핵심 기능

## Phase 2 - 고도화
- [ ] 추가 기능

## Phase 3 - 최적화
- [ ] 성능/UX 개선
```

## 툴 레퍼런스

| 툴 | 언제 | 설명 |
|---|---|---|
| `plan_task` | 작업 시작 시 항상 | 요구사항 분석, 범위 파악 |
| `analyze_task` | 코드베이스 있을 때 | 기존 코드 확인, 중복 방지 |
| `reflect_task` | 복잡한 작업 전 | 분석 재검토, 누락 보완 |
| `split_tasks` | 2단계 이상일 때 | 큰 작업을 서브태스크로 분해 |
| `execute_task` | 각 서브태스크마다 | 특정 태스크 실행 |
| `verify_task` | execute 후 | 완료 여부 검증 |
| `complete_task` | verify 통과 후 | 완료 처리 + 요약 자동 저장 |
| `list_tasks` | 언제든 | 전체 태스크 목록 및 상태 조회 |
| `query_task` | 언제든 | 키워드로 과거 태스크 검색 |
| `get_task_detail` | 언제든 | 특정 태스크 상세 내용 확인 |
| `delete_task` | 언제든 | 미완료 태스크 삭제 (완료된 건 불가) |
