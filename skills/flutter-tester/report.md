# Report Reference

## TEST_REPORT.md 작성 규칙

- 위치: `docs/TEST_REPORT.md` (docs 폴더 없으면 생성)
- 항상 **덮어쓴다** — 이전 리포트 누적 없음
- 타임스탬프 필수
- 버그마다 재현 단계 구체적으로 작성
- CRITICAL/HIGH = 0건일 때만 배포 가능 표시

```bash
mkdir -p docs
# 리포트 작성 (덮어쓰기)
cat > docs/TEST_REPORT.md << 'REPORT'
... 아래 템플릿 내용 ...
REPORT
```
