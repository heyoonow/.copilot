# Scenarios Reference

## 시나리오 ID 체계

```
T-AUTH   : 로그인 / 로그아웃 / 회원가입 / 세션 만료 / 토큰 재발급
T-NAV    : 탭 전환 / 뒤로가기 / 딥링크 / 화면 전환 애니메이션
T-CORE   : 앱의 핵심 기능 (이 앱이 하는 일)
T-FORM   : 입력 → 제출 → 저장 → 목록 반영 전체
T-NET    : 로딩 → 성공/실패 → 재시도 → 오프라인
T-PERM   : 카메라 / 위치 / 알림 권한 요청 & 거부 처리
T-EDGE   : 빈 목록 / 세션 만료 / 중복 제출 / 동시성
T-PERF   : 반복 동작 누적 (리스너 누수, 상태 꼬임, 메모리)
```

---

## 시나리오 작성 포맷

```
[T-XXX-001] 시나리오 이름
- 사용자 행동: 탭/입력/스와이프 등 구체적 순서
- 거쳐야 할 화면: A → B → C
- 최종 확인: 사용자가 실제로 보는 텍스트/UI 상태
- 실패 기준: 이 텍스트/상태가 없으면 버그
- Priority: Critical / High / Medium / Low
```

## T-AUTH — 인증 시나리오

```
[T-AUTH-001] 정상 로그인 → 홈 이동
- 행동: 이메일 입력 → 비번 입력 → 로그인 버튼 탭
- 화면: 로그인 → (로딩) → 홈
- 확인: 홈 화면 노출, BottomNavigationBar 표시
- 실패: 홈 안 뜸, 에러 없이 머무름
- Priority: Critical

[T-AUTH-002] 틀린 비번 → 에러 메시지 → 입력값 유지
- 행동: 이메일 + 잘못된 비번 → 로그인 버튼 탭
- 화면: 로그인 (그대로)
- 확인: 에러 메시지 노출, 홈 미이동, 이메일 입력값 유지
- 실패: 에러 없이 그냥 머무름, 이메일 사라짐
- Priority: Critical

[T-AUTH-003] 빈 필드로 제출 → 유효성 메시지 각 필드 표시
- 행동: 아무것도 입력하지 않고 로그인 버튼 탭
- 확인: 이메일 에러 메시지 + 비번 에러 메시지 각각 노출
- 실패: 에러 없이 API 요청 나감
- Priority: High

[T-AUTH-004] 로그인 성공 후 뒤로가기 → 로그인 화면 복귀 불가
- 행동: 로그인 성공 → 뒤로가기 제스처
- 확인: 로그인 화면으로 돌아가지 않음, 홈 유지
- 실패: 뒤로가기로 로그인 화면 노출
- Priority: High

[T-AUTH-005] 로그아웃 → 로그인 화면 → 재로그인 정상
- 행동: 설정에서 로그아웃 탭 → 로그인 화면 → 다시 로그인
- 확인: 재로그인 후 홈 정상 노출, 이전 사용자 데이터 없음
- Priority: High

[T-AUTH-006] 세션 만료 → 자동 로그아웃 → 만료 메시지
- 행동: 만료된 토큰으로 API 호출 발생
- 화면: 현재 화면 → 로그인
- 확인: '세션이 만료되었습니다' 메시지, 로그인 화면 이동
- Priority: High

[T-AUTH-007] 로딩 중 뒤로가기 → 크래시 없음
- 행동: 로그인 버튼 탭 후 로딩 중 뒤로가기
- 확인: 크래시 없음, 로그인 화면 그대로
- Priority: High
```

## T-NAV — 네비게이션 시나리오

```
[T-NAV-001] 하단 탭 전체 순환
- 행동: 탭1 → 탭2 → 탭3 → ... → 탭1 순서로 전부 탭
- 확인: 각 탭마다 올바른 화면 노출, 탭 선택 상태 표시
- 실패: 탭 탭해도 화면 안 바뀜, 탭 표시 안 됨
- Priority: High

[T-NAV-002] 상세 화면 진입 → 뒤로가기 → 목록 복귀
- 행동: 목록 아이템 탭 → 상세 진입 → 뒤로가기
- 확인: 목록으로 돌아옴, 스크롤 위치 유지
- Priority: High

[T-NAV-003] 같은 버튼 빠르게 5회 탭 → 화면 중복 스택 없음
- 행동: 상세 진입 버튼 빠르게 5회 탭
- 확인: 화면 1개만 스택에 쌓임
- 실패: 동일 화면 5개 push됨
- Priority: Medium

[T-NAV-004] 화면 전환 애니메이션 정상
- 행동: 화면 전환 여러 번 반복
- 확인: 애니메이션 끊김/깜빡임 없음
- Priority: Low
```

## T-CORE — 핵심 기능 시나리오

앱의 메인 기능에 맞게 구체적으로 작성한다. 아래는 CRUD 앱 기준 예시.

```
[T-CORE-001] 아이템 생성 → 목록 즉시 반영
- 행동: 추가 버튼 탭 → 제목 '테스트 생성 아이템' 입력 → 저장
- 화면: 홈 → 폼 → 홈
- 확인: 저장 성공 토스트/스낵바, 목록에 '테스트 생성 아이템' 텍스트 노출
- 실패: 저장됐는데 목록 미반영 (가장 흔한 버그)
- Priority: Critical

[T-CORE-002] 아이템 수정 → 변경사항 반영
- 행동: 아이템 탭 → 제목 '수정된 아이템'으로 변경 → 저장
- 화면: 목록 → 편집 → 목록
- 확인: '수정된 아이템' 텍스트가 목록/상세에 반영
- Priority: Critical

[T-CORE-003] 아이템 삭제 → 목록에서 제거 확인
- 행동: 아이템 삭제 버튼 → 확인 다이얼로그 확인 탭
- 확인: 해당 아이템 목록에서 사라짐, 삭제 성공 피드백
- Priority: Critical

[T-CORE-004] 페이지네이션 / 무한 스크롤
- 행동: 리스트 맨 아래까지 스크롤
- 확인: 추가 데이터 로드됨, 로딩 인디케이터 → 사라짐
- Priority: High

[T-CORE-005] 검색 → 결과 → 초기화
- 행동: 검색어 입력 → 결과 확인 → 검색어 지우기
- 확인: 검색어 포함 결과만 노출, 초기화 후 전체 목록 복원
- Priority: High

[T-CORE-006] 삭제 후 목록 비어있을 때 빈 상태 UI
- 행동: 마지막 아이템 삭제
- 확인: 빈 상태 안내 텍스트 + CTA 버튼 표시
- 실패: 흰 화면, 에러 화면
- Priority: High
```

## T-FORM — 폼 입력 시나리오

```
[T-FORM-001] 필수 필드 빈 채로 제출 → 유효성 에러 각각 표시
- 행동: 필수 필드 비우고 저장 버튼 탭
- 확인: 각 필드 아래 에러 메시지 표시, 저장 미실행
- Priority: High

[T-FORM-002] 입력 → 저장 → 목록에서 동일 텍스트 확인 (절대 빠뜨리지 않는다)
- 행동: 제목 '테스트 입력값 ABC' 입력 → 저장
- 확인: 목록에서 '테스트 입력값 ABC' 텍스트 실제 노출
- 실패: 저장은 됐는데 다른 값이 보이거나 빈 값 보임
- Priority: Critical

[T-FORM-003] 저장 버튼 연속 3회 탭 → 중복 저장 방지
- 행동: 내용 입력 후 저장 버튼 빠르게 3회 탭
- 확인: 아이템 1개만 생성됨, 버튼 비활성화 처리
- 실패: 중복 아이템 2~3개 생성
- Priority: High

[T-FORM-004] 입력 중 뒤로가기 → 미저장 경고 또는 그냥 이탈
- 행동: 내용 입력 후 뒤로가기
- 확인: 앱 정책에 맞는 동작 (경고 다이얼로그 OR 그냥 이탈)
- 실패: 크래시, 빈 아이템 저장
- Priority: Medium

[T-FORM-005] 매우 긴 텍스트 입력 → 레이아웃 깨짐 없음
- 행동: 100자 이상 텍스트 입력 후 저장
- 확인: 오버플로우 없음, 레이아웃 정상
- Priority: Medium

[T-FORM-006] 특수문자 / 이모지 입력 → 정상 저장 및 표시
- 행동: 제목에 '테스트🎉 !@#$' 입력 → 저장
- 확인: 목록에 동일하게 표시
- Priority: Medium
```

## T-NET — 네트워크 시나리오

```
[T-NET-001] API 호출 중 로딩 UI → 완료 후 데이터
- 행동: 데이터 로드 트리거
- 확인: 로딩 중 Shimmer/Indicator 표시 → 완료 후 데이터 렌더링
- 실패: 로딩 표시 없이 바로 데이터, 또는 로딩만 계속
- Priority: High

[T-NET-002] API 실패 → 에러 UI → 재시도 버튼 동작
- 행동: 실패 mock으로 데이터 요청
- 확인: 에러 메시지 + 재시도 버튼 → 탭 후 다시 로딩 시작
- Priority: High

[T-NET-003] 오프라인 → 적절한 안내
- 행동: 네트워크 끊긴 상태에서 앱 사용
- 확인: 오프라인 안내 메시지, 캐시 데이터 표시 (있을 경우)
- Priority: Medium

[T-NET-004] 느린 네트워크 → 중간에 뒤로가기 → 크래시 없음
- 행동: 로딩 중 뒤로가기
- 확인: 크래시 없음, 이전 화면 정상
- Priority: High
```

## T-EDGE — 엣지케이스 시나리오

```
[T-EDGE-001] 빈 목록 상태 → EmptyState UI
- 행동: 데이터 없는 상태로 목록 화면 진입
- 확인: 빈 상태 안내 텍스트 + CTA 버튼 노출
- 실패: 흰 화면, 에러, 이상한 UI
- Priority: High

[T-EDGE-002] 로딩 중 탭 전환 → 크래시 없음
- 행동: API 로딩 중 다른 탭 탭
- 확인: 크래시 없음, 탭 전환 정상, 돌아왔을 때 데이터 정상
- Priority: High

[T-EDGE-003] 동일 화면 10회 진입/이탈 → 상태 꼬임 없음
- 행동: 상세 화면 진입 → 뒤로가기 10회 반복
- 확인: 매번 동일한 UI, 느려지지 않음
- Priority: Medium

[T-EDGE-004] 알림 권한 거부 → 앱 크래시 없음
- 행동: 알림 권한 요청 → 거부
- 확인: 앱 계속 정상 동작, 크래시 없음
- Priority: High

[T-EDGE-005] 카메라/갤러리 권한 거부 → 적절한 안내
- 행동: 이미지 첨부 → 권한 거부
- 확인: 권한 안내 메시지 노출, 크래시 없음
- Priority: Medium
```

## T-PERF — 성능/누적 버그 시나리오

```
[T-PERF-001] 생성/수정/삭제 사이클 5회 반복 → 마지막 상태 정확
- 행동: 아이템 생성 → 수정 → 삭제 사이클 5회 반복
- 확인: 5회 후 상태 정확, 이전 데이터 잔존 없음
- 실패: 반복 후 이상한 상태, 이전 데이터 섞임
- Priority: High

[T-PERF-002] Provider 10회 refresh → 마지막 상태 정상
- 행동: 동일 Provider invalidate 10회 연속
- 확인: 최종 상태 정상, 이전 상태 잔존 없음
- Priority: Medium

[T-PERF-003] 리스트 빠른 스크롤 업다운 반복 → 레이아웃 깨짐 없음
- 행동: 긴 목록에서 위아래 빠른 스크롤 10회
- 확인: 이미지/텍스트 깨짐 없음, 레이아웃 정상
- Priority: Medium
```

## 버그 등급 기준

| 등급 | 기준 | 예시 | 배포 가능? |
|------|------|------|-----------|
| 🔴 CRITICAL | 사용자가 주요 기능 사용 불가 | 로그인 안 됨, 앱 크래시, 저장 안 됨, 데이터 소실 | **절대 불가** |
| 🟠 HIGH | 기능은 동작하지만 결과가 틀림 | 저장됐는데 목록 미반영, 다른 데이터 표시, 중복 저장 | **불가** |
| 🟡 MEDIUM | UI 피드백 누락, 엣지케이스 처리 미흡 | 로딩 표시 없음, 에러 메시지 없음, 권한 거부 처리 미흡 | 판단 필요 |
| 🟢 LOW | 사용성 개선 사항 | 빈 목록 UI 개선, 긴 텍스트 truncate 처리 | 배포 후 처리 가능 |

**배포 기준: 🔴 CRITICAL = 0건, 🟠 HIGH = 0건**
