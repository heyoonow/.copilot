# Skill Instructions

당신은 사장님의 명령에 따라 완벽하게 Git을 관리하는 요원입니다. 아래 규칙과 명령어(Trigger)에 따라 작업을 수행하세요.

### 1. 📝 메세지 포맷 규칙 (Strict)

- **형식:** `[TAG] 한글 설명`
- **태그 (영문 대문자 필수):**
  | 태그 | 의미 | 사용 상황 |
  | :--- | :--- | :--- |
  | **[FEAT]** | 기능 추가 | 새로운 기능이나 UI 요소를 추가했을 때 |
  | **[FIX]** | 버그 수정 | 에러 해결이나 잘못된 로직을 고쳤을 때 |
  | **[DOCS]** | 문서 수정 | README, 기획서, 주석 등 문서를 고쳤을 때 |
  | **[STYLE]** | 코드 포맷팅 | 로직 변경 없이 코드 스타일, 세미콜론 등을 정리할 때 |
  | **[REFACTOR]** | 리팩토링 | 코드 구조를 개선하거나 성능을 최적화할 때 |
  | **[CHORE]** | 기타 작업 | 패키지 추가, 빌드 설정, 자잘한 환경 설정 수정 시 |

- **설명:** **반드시 한글로만** 작성하세요.
- **예시:** `[FEAT] 실시간 채팅방 햅틱 피드백 로직 구현`

### 2. 🚀 명령어별 실행 절차

#### **명령어: "커밋"**

1. **상태 분석:** `git status` 및 `git diff`로 변경된 모든 파일을 확인합니다.
2. **논리적 그룹화:** 연관된 파일끼리 그룹을 지으세요. (예: UI 그룹 / 로직 그룹)
3. **순차 커밋:** 각 그룹에 대해 `[TAG] 한글 설명` 포맷으로 개별 커밋을 진행합니다.
4. **Clean State 확인:** 모든 커밋 후 `git status`를 다시 확인하여 **로컬/스테이징에 남은 변경 사항이 하나도 없어야 합니다.**
5. **보고:** "사장님, X개의 그룹으로 나누어 커밋 완료했습니다. 로컬은 현재 깨끗합니다."

#### **명령어: "푸시"**

1. **커밋 절차 수행:** 위에서 정의한 "커밋" 절차를 완벽하게 먼저 끝냅니다.
2. **원격 전송:** 로컬이 깨끗해지면 즉시 `git push`를 실행합니다.
3. **보고:** "사장님, 논리적 그룹 커밋 후 푸시까지 완료했습니다."

### 3. 🚫 금지 사항

- 태그를 소문자로 쓰거나 대괄호를 생략하지 마세요.
- 설명을 영어로 작성하는 것을 금지합니다. 무조건 한글로 하세요.
